Welcome · v1.0

Your tabs have types now.

Tare replaces Chrome's recency-based tab eviction with typed lifecycles. Critical sessions stay protected. Disposable tabs go first.

The problem

Chrome's Memory Saver uses Least Recently Used eviction. Reasonable as a guess, wrong as a rule.

Chrome Memory Saver

  • Evicts oldest unused tab first
  • No idea what the tab contained
  • Drops your logged-in banking session
  • Keeps your Reddit doom-scroll
  • Forces 2FA re-auth when memory gets tight

Tare

  • Evicts Feed tabs first (○, low priority)
  • Closes Reference tabs after idle (◐)
  • Session tabs (●) stay protected no matter what
  • Auto-drops on real RAM pressure, not tab count
  • 30-second undo window

Three types, one discipline

●

Session

SESSION — always keep

Always keep open. Banking, email, chats, anything where losing your session hurts.

banking, email, calendar, AI chats, work tools, admin panels
◐

Reference

REFERENCE — close after idle

Reference pages you check once. Auto-closed after idle. State saved if you come back.

searches, Wikipedia, Stack Overflow, documentation
○

Feed

FEED — drop first

Scrollable or disposable content. First to close when memory runs low.

Twitter/X, Reddit, Facebook, LinkedIn, news homepages

What it saves

~ 850 MB

Typical memory freed per session

Each Chrome tab holds roughly 85 MB of RAM on average. Discharging 10 tabs frees about 850 MB and stops their background CPU work. Discarded tabs reload instantly on return.

But the real win is that your logged-in sessions stay alive under memory pressure — because your banking tab is typed !, not measured by clicks.

How to test it

Five steps, three minutes.

Open a few sites to see auto-typing

Open new tabs to: mail.google.com, google.com/search?q=test, and reddit.com

The Tare badge changes per tab: ! on Gmail, 1 on search, A on Reddit.

Click the Tare icon

The popup shows your tabs grouped by type, with live system RAM.

Press the shortcut to drop affine

⇧⌘A on Mac, Ctrl+Shift+A on Windows/Linux. Reddit tab gets discharged. Notification shown.

Reopen the popup — green "undo" strip appears. Click undo within 30 seconds to restore.

Override a type manually

Right-click any page → Tare · set tab type → !. Tab gets a "SET" badge confirming manual tag.

Open settings to add your own rule

Click settings at the bottom of the popup. Add a rule for a site you use daily.

What's in v1.0

Real memory pressure

Reads actual system RAM via chrome.system.memory. Auto-drops above 85%.

Editable rules

Add, remove, reorder URL patterns in settings. Export/import as JSON.

Persistent undo

Undo stack survives service worker eviction. 30-second window.

Four keyboard shortcuts

Drop affine, discharge idle, cycle current tab, undo last discharge.

Accessibility

ARIA labels, keyboard navigation, reduced-motion support.

Input validation

All settings and imported configs are schema-validated. No crashing on malformed data.

Honest limits

Chrome Memory Saver still runs alongside. Tare works in parallel. In practice Tare's choices are smarter and get there first.

Protection is advisory. If your system truly runs out of memory, the OS can still kill tabs. Tare reduces this risk but can't eliminate it.

No cross-device sync. Use export/import for now.

For the curious — the theory behind Tare

The four types correspond to Jean-Yves Girard's linear logic (1987): reusable (!), linear (1), affine (A), and neutral (·). This is the same substructural type theory that underpins Rust's ownership system. We use friendlier display labels (Session, Reference, Feed, Other) in the UI, but the underlying discipline is the formal one. If you export a config, you'll see the original symbols there.
